1. 사용한 기술 (1) WebSocket 실시간 양방향 통신을 위한 표준 프로토콜. 클라이언트와 서버 간에 지속적인 연결을 유지하며 데이터를 빠르게 주고받을 수 있음. (2) STOMP (Simple Text Oriented Messaging Protocol) WebSocket 상에서 메시징 기능을 구현하기 위한 프로토콜. 메시지 브로커(Pub/Sub 모델)를 활용하여 효율적인 메시지 전달 가능. 우체통(Topic)이 있다면 잽배원(Publisher)이 신문을 우체통에 배달하는 행위가 있고, 우체통에 신문이 배달되는 것을 기다렸다가 빼서 보는 구독자(Subscriber) pub = 잽배원 , sub = 구독자 1. 채팅방 생성 : pub / sub 구현을 위한 Topic이 생성됨 2. 채팅방 입장 : Topic 구독 3. 채팅방에서 메세지를 송수신 : 해당 Topic으로 메세지를 송신(pub), 메세지를 수신(sub) (3) Spring WebSocket 및 STOMP 지원 Spring Boot에서 WebSocket과 STOMP를 쉽게 구현할 수 있는 모듈 제공. 메시지 핸들러와 컨트롤러로 서버와 클라이언트 간 통신 관리 가능. (4) 프론트엔드 라이브러리 SockJS: WebSocket을 지원하지 않는 브라우저에서도 동작할 수 있도록 하는 폴리필. Stomp.js: STOMP 프로토콜을 사용하여 WebSocket 통신을 처리하기 위한 라이브러리. 2. 구현 구조 Backend: Spring WebSocket WebSocket Config 설정 WebSocket 엔드포인트 정의. STOMP 메시지 브로커 설정. Controller 및 Service 메시지 핸들러 작성 (예: 특정 채팅방으로 메시지 전송). STOMP를 통해 메시지 라우팅. Frontend SockJS와 Stomp.js 사용 WebSocket 연결 초기화. 메시지 송수신 구현.
